使用github仓库托管静态博客

2019-05-21 11:27:00 | 技术类

前几天我们已经简单介绍了hexo博客的搭建过程，现在我们可以换一下玩法，把博客丢到github上托管了。

如果没有github账号的朋友可以先去github官网申请一个账号。(这个过程太简单了，我们就跳过)

  • 然后点击右上角的 + 号 ，选 New repository创建一个新的项目仓库

  • 创建一个与自己github名字 同名.github.io 的项目
  • 然后在个人主页就可以看到这个仓库了
配置ssh连接Github
  • 在命令行输入 ssh-keygen 然后一路回车，直到看到类似界面
    (如果提示命令不存在，要先安装一下或者配置一下路径)

  1. 把ssh的key配置到Github，先 vi ~/.ssh/id_rsa.pub 把里面的内容复制

  2. 在github页面里点击右上角的头像，选择setting

  3. 找到 SSH and GPG keys 再点击 New SSH key

  4. 填写你的ssh-Key的标题和内容

  5. 点击 Add SSH Key 保存

  6. 在服务器输入ssh -T git@github.com 测试SSH是否设置成功

  7. 配置_config.yml 使用ssh的方式连接到github仓库
    vi _config.yml

  1. 依次执行

就会看到hexo同步到github仓库的信息，如果有错误信息，就是github账号信息没有配置成功，就填写一下 ~/.gitconfig 里面的github账号信息。

  • 然后浏览器访问 https://你的github名字.github.io 就能看到我们在github托管的博客了。
